# Changelog

> EVO Method é um fork do [BMad Method](https://github.com/bmadcode/bmad-method), com melhorias, extensões e adaptações feitas pela equipe da Evolution. As versões abaixo refletem mudanças realizadas a partir do fork da base 6.0.4 do BMad.

---

## [1.0.6] - 2026-03-09

### 🎁 Features

* Atualiza mensagem de início da instalação para o lançamento do EVO Method V6

---

## [1.0.5] - 2026-03-09

### 🎁 Features

* Adiciona skill `set-feature` para definir e alternar o contexto de feature ativa, organizando artefatos em subpastas por feature

---

## [1.0.4] - 2026-03-09

### 🎁 Features

* Adiciona skills nativas do Claude Code para os agentes EVO instalados

---

## [1.0.3] - 2026-03-09

### ♻️ Refactoring

* Workflows passam a escrever conteúdo diretamente no arquivo de saída ao invés de apresentar ao usuário
* Padrão de menu de colaboração atualizado de A/P/C para A/P/C/R (adiciona opção Rewrite)

---

## [1.0.2] - 2026-03-09

### 🎁 Features

* Refatora workflows para suportar contexto de feature ativa (`active_feature`) com organização de artefatos em subpastas
* Adiciona diretiva de conciseness nos workflows `dev-story` e `quick-dev`

### ♻️ Refactoring

* Escopo do pacote npm atualizado para `@evoapi/evo-method`

---

## [1.0.0] - 2026-03-09

### 🎁 Features

* Fork do BMad Method 6.0.4 — rebrand completo para **EVO Method** (Evolve, Validate, Orchestrate)
* Atualização de repositório, autor e referências para EvolutionAPI
* Versão inicial publicada como `@evoapi/evo-method`
